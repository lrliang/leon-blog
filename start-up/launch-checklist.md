# 启动清单：可执行任务列表

> 按照优先级和依赖关系排列，逐项完成即可启动

---

## 阶段一：基础设施搭建（Week 1）

### 1.1 品牌资产准备

- [ ] **确定账号名称**
  - 建议风格：真名 + 标签 或 有辨识度的品牌名
  - 示例：「Leon 的代码手记」「独立开发日志」
  - 要求：各平台统一

- [ ] **设计头像**
  - 选项 A：专业证件照/生活照
  - 选项 B：品牌 Logo（可用 Figma/Canva 设计）
  - 要求：清晰、专业、有辨识度

- [ ] **撰写个人简介**
  - 准备 3 个版本：
    - 长版（200 字，用于博客/公众号）
    - 中版（100 字，用于掘金）
    - 短版（50 字，用于小红书/抖音）
  - 包含：身份标签 + 在做的事 + 联系方式

- [ ] **准备联系方式**
  - 公开邮箱（推荐 Gmail 或自有域名邮箱）
  - 微信号（决定是否公开个人微信）
  - 可选：创建公众号/知识星球

### 1.2 平台账号注册

- [ ] **微信公众号**
  - 注册类型：个人订阅号
  - 完成认证（如需要）
  - 设置菜单栏：往期精选 / 关于我 / 联系方式
  - 配置自动回复

- [ ] **个人博客**
  - 技术选型：Hugo / Next.js / mdBook
  - 部署平台：GitHub Pages / Vercel / Cloudflare Pages
  - 域名注册（可选，建议购买）
  - 基础页面：首页 / 关于 / 文章列表

- [ ] **掘金**
  - 注册账号
  - 完善个人资料
  - 创建专栏：Rust 实战 / 独立开发日志

- [ ] **小红书**
  - 注册账号（需手机号）
  - 完善资料、上传头像
  - 发布首条笔记（自我介绍）

- [ ] **其他平台（可选）**
  - GitHub：完善 Profile README
  - 即刻：加入相关圈子
  - Twitter/X：技术出海准备

### 1.3 工具准备

- [ ] **写作工具**
  - Typora / Obsidian / VS Code（Markdown 写作）
  - 安装排版插件

- [ ] **图片工具**
  - Figma（架构图、设计稿）
  - Excalidraw（手绘风格图）
  - Carbon.now.sh（代码截图美化）
  - Canva（封面图、社交图片）

- [ ] **素材管理**
  - 创建素材库文件夹结构
  - 截图工具配置（系统自带 / Snipaste）
  - 录屏工具：OBS / ScreenFlow

- [ ] **发布工具**
  - 公众号：Markdown 编辑器（墨滴 / 壹伴）
  - 多平台发布：自动同步或手动发布流程

---

## 阶段二：首批内容创作（Week 1-2）

### 2.1 内容规划

- [ ] **确定首批发布内容**
  - 公众号：2-3 篇
  - 掘金：1-2 篇
  - 小红书：3-5 篇
  - 博客：同步公众号

- [ ] **制定内容日历**
  - 确定发布时间（如每周三晚 8 点）
  - 规划未来 4 周内容

### 2.2 公众号首发内容

- [ ] **文章 1：自我介绍**
  - 标题：《你好，我是 XXX，一个独立开发者》
  - 内容：个人背景、为什么做这个号、读者能得到什么

- [ ] **文章 2：项目启动**
  - 标题：《为什么我要用 Rust 开发第一个桌面应用》
  - 内容：项目缘起、技术选型、开发目标

- [ ] **文章 3：技术教程**
  - 标题：《Dioxus 入门：搭建你的第一个 Rust 桌面应用》
  - 内容：环境配置、项目初始化、Hello World

### 2.3 掘金首发内容

- [ ] **文章 1：Rust 入门**
  - 标题：《从 JavaScript 到 Rust：思维转换指南》
  - 内容：对比学习、核心概念、快速上手

### 2.4 小红书首批内容

- [ ] **笔记 1：自我介绍**
  - 内容：「👋 你好，我是一个独立开发者...」

- [ ] **笔记 2：项目启动**
  - 内容：「开始用 Rust 开发第一个桌面应用」

- [ ] **笔记 3：工具分享**
  - 内容：「我的开发工具栈分享」

### 2.5 博客内容

- [ ] 同步公众号文章
- [ ] 添加 asset-light 项目介绍页
- [ ] 配置 SEO 基础设置

## 阶段三：正式发布（Week 2-3）

### 3.1 内容发布

- [ ] **公众号首发**
  - 检查排版
  - 预览效果
  - 群发推送
  - 朋友圈分享

- [ ] **博客同步**
  - 发布文章
  - 检查链接
  - 社交分享

- [ ] **掘金发布**
  - 调整格式
  - 添加标签
  - 投稿活动（如有）

- [ ] **小红书发布**
  - 按节奏发布
  - 回复评论
  - 观察数据

### 3.2 初期推广

- [ ] **社群分享**
  - 技术社群（Rust 中文、Dioxus 社区）
  - 独立开发者社群
  - 朋友圈

- [ ] **技术社区参与**
  - 在相关文章下评论
  - 回答 Rust/Dioxus 相关问题
  - 参与讨论

- [ ] **SEO 优化**
  - 提交博客到搜索引擎
  - 添加 sitemap
  - 内链优化

## 阶段四：建立节奏（Week 3-8）

### 4.1 内容持续产出

- [ ] 保持周更节奏（公众号 + 博客）
- [ ] 小红书保持 2-3 篇/周
- [ ] 掘金每周同步
- [ ] 建立素材积累习惯

### 4.2 数据追踪

- [ ] **设置数据看板**
  - 公众号：后台数据
  - 博客：Google Analytics / Umami
  - 掘金：个人主页数据
  - 小红书：创作者中心

- [ ] **周度复盘**
  - 哪些内容表现好？
  - 读者反馈是什么？
  - 下周如何调整？

### 4.3 互动与社区

- [ ] 及时回复评论
- [ ] 关注同领域创作者
- [ ] 参与技术社区讨论
- [ ] 加入博主互推群（可选）

## 阶段五：视频启动（Week 6-8，可选）

### 5.1 视频准备

- [ ] **硬件准备**
  - 麦克风（推荐 USB 麦克风）
  - 摄像头（可选，可用手机）
  - 补光灯（可选）

- [ ] **软件准备**
  - 录屏：OBS
  - 剪辑：剪映 / Final Cut
  - 字幕：剪映自动字幕

### 5.2 平台开通

- [ ] 视频号开通
- [ ] 抖音账号设置
- [ ] B站（可选）

### 5.3 首批视频

- [ ] **视频 1：自我介绍**
  - 时长：1-2 分钟
  - 内容：我是谁、在做什么、为什么关注我

- [ ] **视频 2：工具演示**
  - 时长：30-60 秒
  - 内容：Cursor 使用技巧 / 开发效率

- [ ] **视频 3：项目展示**
  - 时长：1-2 分钟
  - 内容：asset-light 功能演示
