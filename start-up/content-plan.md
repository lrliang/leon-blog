## 4. 创作流程

### 4.1 单篇内容创作流程

```
1. 选题确认
   └── 从选题库选择 or 热点触发

2. 素材准备（1h）
   ├── 收集相关资料
   ├── 梳理大纲
   └── 准备代码示例

3. 初稿写作（2-3h）
   └── 一气呵成，不求完美

4. 修改润色（1h）
   ├── 逻辑检查
   ├── 语言优化
   └── 配图制作

5. 多平台适配（0.5h）
   ├── 公众号排版
   ├── 掘金格式调整
   └── 博客发布

6. 发布上线
   ├── 主平台发布
   ├── 社群分享
   └── 评论区互动
```

### 4.2 素材积累习惯

**日常积累：**
- 开发过程中随手截图
- 遇到问题记录解决方案
- 阅读优质内容时做笔记
- 收集有启发的观点

**素材库组织：**
```
素材库/
├── 截图/
│   ├── 代码截图/
│   ├── UI 截图/
│   └── 报错截图/
├── 笔记/
│   ├── 技术笔记/
│   ├── 阅读笔记/
│   └── 灵感记录/
```

## 5. 爆款内容特征

### 5.1 高传播内容特征

| 特征 | 说明 | 示例 |
|------|------|------|
| **实用性强** | 读完能立即用 | 《10 个 Cursor 快捷键》 |
| **门槛适中** | 入门友好但有深度 | 《Rust 入门第一步》 |
| **有共鸣** | 戳中痛点或引发思考 | 《程序员 35 岁焦虑破解》 |
| **有争议** | 观点鲜明，引发讨论 | 《为什么我放弃了 TypeScript》 |

| **数据化** | 有数据支撑 | 《用 AI 后代码效率提升 300%》 |

## 6. 内容质量检查清单

### 6.1 发布前检查

- [ ] 标题是否吸引人？
- [ ] 开头是否能抓住注意力？
- [ ] 逻辑是否清晰？
- [ ] 代码是否可运行？
- [ ] 配图是否清晰美观？
- [ ] 是否有错别字？
- [ ] 结尾是否有 CTA（引导关注/评论）？
- [ ] 各平台格式是否正确？

### 6.2 内容评分维度

| 维度 | 权重 | 评分标准 |
|------|------|---------|
| 实用价值 | 30% | 读者能获得什么 |
| 内容深度 | 25% | 信息密度与洞察 |
| 可读性 | 20% | 语言流畅、结构清晰 |
| 原创性 | 15% | 独特视角与见解 |
| 视觉呈现 | 10% | 排版、配图、代码格式 |
