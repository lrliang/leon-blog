## 2. 微信公众号

### 2.1 平台定位

| 维度 | 定义 |
|------|------|
| **角色** | 内容中枢、品牌主阵地 |
| **目标** | 沉淀深度内容、建立专业形象、转化私域 |
| **优先级** | P0（必须做好） |

### 2.2 账号设置

**账号名称建议：**
- 「Leon 的代码手记」
- 「独立开发日志」
- 「AI 开发效能局」
- 或使用真名 + 标签

**头像：** 专业头像或品牌 Logo

**简介模板：**
```
🔹 Rust 开发者 | AI 效能顾问 | 独立开发者
🔹 专注：AI 辅助开发 × 桌面应用 × 开发效能
🔹 在做：asset-light 个人资产管理工具
🔹 可提供：企业软件开发效能咨询

合作/咨询：微信 xxx | 邮箱 xxx@xxx.com
```

### 2.3 内容策略

**内容类型分布：**

| 类型 | 占比 | 示例 |
|------|------|------|
| 深度技术 | 30% | 《Rust 所有权彻底搞懂》《Dioxus 状态管理实战》 |
| 项目实战 | 30% | 《从 0 到 1 构建桌面应用》《asset-light 开发全记录》 |
| 方法论 | 20% | 《如何用 AI 提升 3 倍开发效率》《独立开发者的产品思维》 |
| 工具效率 | 10% | 《我的开发工具栈 2025》《Cursor 高效使用技巧》 |
| 个人成长 | 10% | 《独立开发第一年的收获》《月度复盘与反思》 |

**发布节奏：**
- 周更 1-2 篇（质量 > 数量）
- 固定发布日（如每周三晚 8 点）
- 重要节点加更（产品发布、里程碑等）

**排版规范：**
- 使用统一的品牌色
- 代码块使用图片（提升阅读体验）
- 适当使用 emoji 增强可读性
- 文末固定模块：个人介绍 + 往期推荐

### 2.4 增长策略

1. **社群互推**：加入技术博主互推群
2. **优质转载**：允许白名单转载，扩大传播
3. **活动参与**：参加技术活动，分享 PPT 引流
4. **SEO 联动**：公众号文章同步博客，长尾流量引流

## 3. 个人博客

### 3.1 平台定位

| 维度 | 定义 |
|------|------|
| **角色** | 技术资产库、SEO 长尾入口 |
| **目标** | 完整内容沉淀、搜索引擎优化、专业形象展示 |
| **优先级** | P0 |

### 3.2 技术选型建议

| 方案 | 适合场景 | 推荐度 |
|------|---------|--------|
| **Hugo + GitHub Pages** | 快速部署、Markdown 原生 | ⭐⭐⭐⭐⭐ |
| **Next.js + Vercel** | 想要更多定制 | ⭐⭐⭐⭐ |
| **mdBook** | 与项目文档统一 | ⭐⭐⭐⭐ |
| **Notion + Super** | 懒人方案 | ⭐⭐⭐ |

### 3.3 内容规划

**核心板块：**

```
博客首页
├── 📝 技术文章
│   ├── Rust 学习系列
│   ├── Dioxus 实战系列
│   ├── AI 开发效能
│   └── 工具与效率
│
├── 🚀 项目展示
│   ├── asset-light
│   └── 其他项目
│
├── 📚 学习笔记
│   ├── 读书笔记
│   └── 课程笔记
│
├── 💭 思考随笔
│   └── 复盘与反思
│
└── 👋 关于我
    ├── 个人简介
    └── 联系方式
```

### 3.4 SEO 优化要点

1. **标题优化**：包含核心关键词（如 "Rust 教程"、"Dioxus 入门"）
2. **URL 结构**：语义化 URL（如 `/rust/ownership-explained`）
3. **Meta 描述**：每篇文章写好描述
4. **内链建设**：文章间相互链接
5. **外链获取**：在技术社区分享，获取反向链接

## 4. 掘金

| **目标** | 获取技术圈流量、建立技术影响力 |

### 4.2 账号设置

**专栏设置：**
- 创建 "Rust 实战" 专栏


### 4.3 内容策略

**适合掘金的内容类型：**

| 类型 | 热度 | 示例标题 |
|------|------|---------|
| 入门教程 | ⭐⭐⭐⭐⭐ | 《Rust 入门：从 JavaScript 开发者视角》 |
| 实战项目 | ⭐⭐⭐⭐⭐ | 《用 Rust + Dioxus 开发我的第一个桌面应用》 |

**发布技巧：**
- 标题党适度（但内容要扎实）
- 首图精心设计（吸引点击）
- 合理使用标签（增加曝光）
- 参与官方活动（如月更挑战）

### 4.4 增长策略

1. **投稿周刊**：优质文章投稿掘金周刊
2. **参与活动**：月更挑战、年度征文等
3. **评论互动**：积极回复评论，参与讨论
4. **交叉引流**：文末引导关注公众号

## 5. 小红书

### 5.1 平台定位

| 维度 | 定义 |
|------|------|
| **角色** | 泛人群触达、轻量内容分发 |
| **目标** | 破圈、触达非技术用户、个人 IP 展示 |
| **优先级** | P1 |

### 5.2 账号设置

**账号名称：** 与公众号保持一致或用真名

**简介模板：**
```
👨‍💻 程序员 | 独立开发者
🛠️ 正在开发个人资产管理工具
✨ 分享开发日常 | 效率工具 | 程序员生活
📱 公众号同名
```

### 5.3 内容策略

**适合小红书的内容类型：**

| 类型 | 示例 |
|------|------|
| 开发日常 | 「今天写了 500 行代码，终于把这个 bug 修好了 🎉」 |
| 工具推荐 | 「程序员必备的 5 个效率工具」 |
| 桌面/工作环境 | 「我的极简开发桌面分享」 |
| 知识卡片 | 「一图看懂 Rust 所有权」 |
| 产品展示 | 「我开发的个人资产管理工具长这样」 |
| 程序员生活 | 「独立开发者的一天」 |

**内容形式：**
- 图文笔记（1-9 张图）
- 轮播图（知识卡片）
- 短视频（15-60 秒）

**发布节奏：**
- 日更或隔日更
- 早 8 点 / 午休 / 晚 8 点发布

### 5.4 风格调性

- 真实、轻松、有人味
- 适当展示"幕后"（工作台、开发中的界面、bug 日常）
- 不要太"技术味"，降低理解门槛
- 善用 emoji 和口语化表达

## 6. 抖音 / 视频号

### 6.1 平台定位

| 维度 | 定义 |
|------|------|
| **角色** | 视频引流、扩大影响力边界 |
| **目标** | 破圈传播、吸引潜在咨询客户 |
| **优先级** | P2（图文稳定后启动） |

### 6.2 账号设置

保持与其他平台一致的账号名、头像、简介。

### 6.3 内容策略

**视频类型规划：**

| 类型 | 时长 | 示例 | 适合平台 |
|------|------|------|---------|
| 技术科普 | 1-3 分钟 | 《什么是 Rust？为什么它这么火》 | 视频号 |
| 工具演示 | 30-60 秒 | 《这个 AI 工具让我写代码快 3 倍》 | 抖音 |
| 开发 Vlog | 2-5 分钟 | 《独立开发者的一周》 | 视频号 |
| 产品展示 | 1-2 分钟 | 《我开发的资产管理工具》 | 双平台 |
| 干货分享 | 1-3 分钟 | 《程序员提升效率的 5 个技巧》 | 双平台 |

**制作要点：**
- 前 3 秒抓住注意力
- 节奏快、信息密度高
- 字幕必须加
- 竖屏 9:16 为主

### 6.4 发布节奏

- 起步期：周更 1-2 条
- 稳定期：每周 3-5 条
- 固定时间发布（建立用户预期）

## 7. 跨平台联动策略

### 7.1 内容复用矩阵

```
┌─────────────────────────────────────────────────────────────┐
│                     一篇深度文章                             │
│              《用 Rust 开发个人资产管理工具》                  │
└───────────────────────────┬─────────────────────────────────┘
                            │
            ┌───────────────┼───────────────┐
            ▼               ▼               ▼
    ┌───────────────┐ ┌───────────────┐ ┌───────────────┐
    │   公众号原文   │ │   博客全文    │ │   掘金同步    │
    │   + 排版美化   │ │   + SEO 优化  │ │   + 互动维护  │
    └───────────────┘ └───────────────┘ └───────────────┘
            │               │               │
            └───────────────┼───────────────┘
                            │
            ┌───────────────┼───────────────┐
            ▼               ▼               ▼
    ┌───────────────┐ ┌───────────────┐ ┌───────────────┐
    │   小红书       │ │   短视频      │ │   微博/即刻   │
    │   核心要点图文 │ │   1分钟精华版 │ │   金句摘录    │
    └───────────────┘ └───────────────┘ └───────────────┘
```

### 7.2 引流路径设计

```
抖音/小红书 (泛流量)
        │
        ▼  "关注公众号获取完整教程"
微信公众号 (留存沉淀)
        │
        ▼  "加微信进独立开发者群"
私域微信 (深度转化)
        │
        ├──▶ 咨询服务
        └──▶ 产品购买
```

### 7.3 账号互相关联

- 各平台简介中包含其他平台入口
- 内容末尾统一引导语
- 私信自动回复引导关注其他平台
