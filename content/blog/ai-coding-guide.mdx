## TL;DR

- AI 辅助编程不是替代程序员，而是成为你的超级助手
- 掌握 Prompt Engineering 是基础，理解 Agent 是进阶
- 道-法-术-器：从理念到工具，系统提升效率

## 为什么 AI 编程很重要

在 2024 年，AI 辅助编程已经从"可选"变成了"必备"技能。

<Callout type="tip" title="核心观点">
AI 不会取代程序员，但会用 AI 的程序员会取代不会用的。
</Callout>

## Prompt Engineering 基础

### 清晰的指令

好的 Prompt 应该包含：

1. **角色定义**：你希望 AI 扮演什么角色
2. **任务描述**：具体要完成什么
3. **约束条件**：有什么限制或要求
4. **输出格式**：期望的结果形式

### 代码示例

```typescript
// 一个简单的 React 组件
function UserCard({ name, avatar }: { name: string; avatar: string }) {
  return (
    <div className="flex items-center gap-4 p-4 rounded-lg border">
      <img src={avatar} alt={name} className="w-12 h-12 rounded-full" />
      <span className="font-medium">{name}</span>
    </div>
  );
}
```

## Agent 工作流

Agent 是 AI 编程的进阶形态，它能够：

- 自主规划任务
- 调用外部工具
- 迭代优化结果

<Callout type="warning" title="注意">
Agent 虽然强大，但也需要人类的监督和引导。
</Callout>

## 实践建议

1. 从小任务开始，逐步扩大 AI 的参与范围
2. 保持代码审查习惯，AI 生成的代码也需要 review
3. 持续学习，AI 工具在快速迭代
